## Bug1：API错误率显示50% 🔴

### 现象
Dashboard显示 "API错误率: 50.0%，阈值: <2%，今日6次调用中3次失败"

### 根因分析
**代码层根因：`ReportsService.getOverview()` 从未计算 `apiErrorRate` 字段**

- 旧版 Express `server.js` 有硬编码种子数据（固定3个API错误），迁移到 NestJS 时遗漏了此计算
- 前端 `AdminDashboard.vue` 调用 `/admin/dashboard` 端点，但该端点**根本不存在于后端**
- 后端仅有 `/reports/overview` 端点，且返回的字段不包含 `apiErrorRate`
- 前端 fallback 逻辑在 API 调用失败时可能使用默认值或缓存值

**数据流断裂：**
```
前端期望: GET /admin/dashboard → { apiErrorRate: "2.50" }
后端实际: 端点不存在 → 404 → 前端 catch 使用默认值
```

### 修复方案 ✅ 已实施
1. 在 `ReportsService` 中新增 `getAdminDashboard()` 方法
2. 从 `ai_call_records` 表实时计算：`failed数量 / 总调用数 × 100`
3. 在 `ReportsController` 中添加 `GET /admin/dashboard` 端点
4. 返回格式：`apiErrorRate: "2.50"` (百分比字符串)

## Bug7：订单管理数据列错位 🔴

### 现象
订单管理页面的表格列数据错位，显示 undefined 或空白

### 根因分析
**代码层根因：`PaymentRecords.vue` 使用了错误的字段名映射**

| 前端使用字段 | 后端实际字段 | 状态 |
|-------------|------------|------|
| `order.orderNo` | `transactionNo` | ❌ 不匹配 |
| `order.coins` | `coinAmount` | ❌ 不匹配 |
| `order.payMethod` | `payChannel` | ❌ 不匹配 |
| `status: 'completed'` | `status: 'success'` | ❌ 枚举值错误 |

**额外发现6个严重问题：**

1. **API端点错误**：`adminApi.getOrders()` 调用 `/payment/orders`，但该端点有 `@CurrentUser` 守卫，只返回当前用户的订单，管理员看不到所有订单
2. **确认订单端点不存在**：`adminApi.confirmOrder()` 调用 `PUT /payment/orders/:id/confirm`，后端无此端点
3. **响应结构不匹配**：前端检查 `data.code === 0`，但支付控制器不返回 `code` 字段
4. **数据路径错误**：即使 API 成功，`data.data` 是 `{ items, total, page, pageSize }` 对象，不是数组
5. **使用原生 fetch**：`PaymentRecords.vue` 用 `fetch('/api/v1/...')` 而非共享 API 服务，绕过认证和基础URL配置
6. **状态枚举错误**：前端用 `completed`，后端枚举是 `['pending', 'success', 'failed', 'refunded']`

### 修复方案
**需9号前端修复工程师配合：**
1. `PaymentRecords.vue` 字段名修正：`orderNo→transactionNo`, `coins→coinAmount`, `payMethod→payChannel`
2. 状态值修正：`completed→success`
3. 改用 `adminApi` 共享服务
4. 后端需新增管理员专用订单列表端点 `GET /admin/orders`

## Bug13：用户增长率13283.3%异常 🔴

### 现象
Dashboard显示 "用户增长率: 13283.3%"

### 根因分析
**代码层根因：`/admin/dashboard` 端点不存在 + `todayRegistrationsChange` 字段缺失**

前端计算逻辑：
```javascript
regChange.value = d.todayRegistrationsChange ?? regChange.value
// regChange 初始值 = 13283.3 (硬编码默认值)
```

**数据流断裂：**
1. 前端调用 `GET /admin/dashboard` → 404 Not Found
2. catch 块捕获错误，但 `regChange.value` 保持初始默认值 13283.3
3. 即使端点存在，旧版 `getOverview()` 也不返回 `todayRegistrationsChange` 字段

**根本问题：**
- 后端缺少 `/admin/dashboard` 聚合端点
- 无昨日注册用户数对比数据
- 前端默认值未设为0

### 修复方案 ✅ 已实施
1. 新增 `getAdminDashboard()` 方法，计算：
   - `newUsersToday`：今日注册用户数
   - `newUsersYesterday`：昨日注册用户数
   - `todayRegistrationsChange`：`(today - yesterday) / yesterday × 100`
2. 返回 `realtimeStats` 数组格式（前端期望的结构）
3. 所有变化率字段都有正确计算值
4. 前端无需修改（字段名已匹配）
